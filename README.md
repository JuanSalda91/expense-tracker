# Expense Tracker Page Project
Case Study 1 – Front-End Development  
Built with HTML, CSS, and JavaScript

---

## Overview

This project fulfills the requirements for **Case Study 1: Interactive Landing Page**, where the goal is to build a modern, responsive landing page for a product of the student's choice.  

**My chosen product:** A Budget Tracker Web Application.

The landing page serves as a promotional introduction to the budgeting tool, followed by the fully functional Budget Tracker interface that allows users to input transactions, categorize expenses, and view data visualizations.

This project demonstrates:
- Responsive web design  
- Clean UI layout  
- JavaScript interactions  
- Modal popups  
- A multi-section landing page  
- A functional budgeting mini-application  

---

## Features – Landing Page (Case Study Requirements)

### Required Features
- **Navbar** with at least three navigation links  
- **Hero section** with title, subtitle, and call-to-action button  
- **Features section** with 3–4 product highlights  
- **CTA section** that triggers JavaScript actions  
- **Responsive layout** using Flexbox and Grid  
- **At least 3 Git commits** (start, feature, final) using branches  

### JavaScript Interaction Requirements
The CTA uses JavaScript to trigger interactions such as:
- Modal popup  
- Smooth scrolling  
- Alert message  

### Stretch Goal Features
- Animated button hover effects  
- Responsive hamburger menu  
- Modal with fade-in animation  
- Prepared for optional light/dark mode toggle  

---

## Features – Budget Tracker Application UI

After the landing page, the page transitions into the Budget Tracker App, which includes:
- Total Income / Total Expense / Balance summary cards  
- Add Transaction form (type, amount, category, date, description)  
- Auto-populated category list  
- Real-time data updates  
- Pie chart visualization (Chart.js)  
- Complete transaction history list  
- Empty state for no data
